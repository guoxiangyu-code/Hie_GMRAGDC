  因此，当前 Zero 不能作为主论文创新，只能作为“判空 operating point 可被独立重校准”的现象证据。

  ## 三、EaTR、QD 和严格训练协议

  ### EaTR

  Q/D/C/full 的数字均正确，但“same frozen parent”不准确：

  - 子模型从同一 parent checkpoint 初始化；
  - shared 参数仍以 3e-6 更新，新模块以 3e-5 更新；
  - 缺少不加新模块、但继续相同步数训练的 continued control。

  普通 query bootstrap 下，Q/D 的 G@3 增益区间为正，但 mAP 区间均跨 0；再考虑单 seed、continued-training 混杂，目前只能说“有判空/GMR 信号”，不能宣称 localization 改善已经成立。

  D 也不只是 “Dual gate”，而是包含双路记忆、DQA/EOS 辅助损失和残差门的完整 dual-grounding adapter。

  ### CG 的 strict 例外

  Moment、EaTR、QD、Flash 的核心 null masking 基本正确。但 CG 的 global_rep_tokens 没有按 positive mask 过滤，null 样本仍进入正交辅助损失，见methods/cg_detr_gmr/model.py:979。

  所以 CG 当前只能写“null 不进入监督 VMR loss”，不能写成完全 strict；其负结果也不宜作为强因果证据。

  ## 四、Learned P 是最值得继续的点，但尚未解决 multi-moment

  三个真正独立的 selector seed 上，固定 Top-3 的 mAP 改善为：

  - +0.57；
  - +0.31；
  - +0.39；
  - 三个独立 seed 的平均改善约 +0.42，而不是 PDF 按五组加权后的 +0.48。

  优点是：

  - P 的训练标签来自 train GT，没有使用 test label；
  - Direct Top-3 和 Learned Top-3 使用相同候选、相同 K，这是目前最干净的比较；
  - 三个独立 selector seed 的方向一致。

  但主要问题是：

  - mR+@3 从 0.44 降到 0；
  - mIoU+@3 也下降；
  - 78 个 selector 配置在同一 validation 上选择并在同一集合报告；
  - 还没有与 Hard-NMS、complete-link 在完全相同候选输入上比较。

  因此它证明的是“排序/去重可能提高总体 mAP”，尚未证明“解决多个独立事件的覆盖”。

  另外，所谓 Soft Count 的五个最佳配置全部 count_prior_weight=0；实际贡献来自 variable-length early stopping，而不是 count posterior。应改名，或补做 β>0 的独立确认实验。

  ## 六、发文章可能性排序

  ### 1. Coverage-aware event set decoding：潜力最高

  建议把主故事集中为：

  > 独立空集验证解决 N=0，事件关系与覆盖优化解决 N>1。

  不要继续围绕 Q+D+C+Z+P 的模块堆叠。真正有论文潜力的是把 P 改造成面向 unique-event coverage 的集合预测目标，例如 unmatched-GT coverage、facility-location 或 DPP 风格目标。

  当前潜力高，但还不能投稿 full paper，因为 P 仍伤害 mR+，Z 也没有稳定和完全解耦。

  ### 2. 现有 GMR 论文修订/扩充：近期最成熟

  Flash from-scratch 复现、EaTR/QD/CG 的适用边界、continued-control 审计，很适合用于现有工作的 revision、supplement 或期刊扩展。

  ### 3. GMR 指标与协议审计：适合短文/Workshop

  验证集中 null 占 210/465=45.16%，全判空模型天然能得到约 45.16 的 aggregate G。旧 QD 的 G≈46 很可能掩盖了近似 all-empty collapse。

  可以围绕以下内容形成短文：

  - aggregate G 的 collapse 风险；
  - null/single/multi macro-balanced G；
  - strict null supervision；
  - continued control 如何改变模块结论；
  - threshold-free rejection–localization 曲线。

  但必须增加第二领域和新评估指标，否则容易被视为勘误或补充实验。

  ### 4. Q/D/C 非可加性：适合作为分析章节

  Counter/full HieA2M 在 EaTR、QD、CG 中失败，Q/D 跨骨干效果高度不一致。这是有价值的负结果，但目前不适合单独发 full paper。

  ## 七、建议的后续实验顺序

  ### P0：先修正因果和验证协议

  1. 按视频或比赛划分 calibration/confirmation；test 继续保持 SEALED。
  2. Moment 重做 matched 三行：continued parent、Z-only、B+Z cascade。
  3. 实现真正 Z(no Counter)，并加入参数量匹配的控制头。
  4. QD/EaTR 固定相同 parent、prefix LR、优化步数、正样本曝光和 checkpoint 规则，再跑 B/Q/D/Q+D。
  5. 将 PDF 中 “frozen parent”“独立 Zero”“Soft Count”“五组稳定复现”等措辞全部修正。

  ### P1：把 P 升级成真正的 event coverage 方法

  在完全相同 raw candidates 和 K 下比较：

  - Direct Top-K；
  - Hard-NMS；
  - Complete-link；
  - 当前 learned P；
  - coverage-aware P；
  - oracle unique-GT coverage。

  新增指标：

  - duplicate rate；
  - unique-GT recall/coverage；
  - mR+、mIoU+；
  - pairwise AUPRC；
  - 时间相邻但属于不同事件的 hard negatives。

  只有当 P 同时提高 mAP 和 multi-event coverage，才值得作为新主方法。

  ### P2：形成论文级证据

  - Moment、EaTR、QD 三个骨干；
  - 2023/2024/2025 三个完整 backbone seeds，而不是只重训 head；
  - 增加 Gymnastics-GMR 或第二领域；
  - 按视频/比赛做 cluster bootstrap；
  - Flash 等当前任务自然结束后增加 compute/data-matched control；
  - 所有 raw/NMS 结果成对报告；
  - 最终模型冻结后再进行一次 blind test。

  总体建议：近期先整理为现有 GMR 工作的强化证据；新 full paper 则集中投入“coverage-aware event set decoding”。当前 Zero、Soft Count、Quality、Dual 或 Flash-Q 都不适合单独作为主创新。
