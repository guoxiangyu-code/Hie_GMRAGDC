Task

Why Generalized Moment Retrieval?

Conventional video moment retrieval usually assumes that every query has exactly one matching temporal segment. Real retrieval scenarios are broader: an event may be absent, occur once, or repeat multiple times in the same video.

Generalized Moment Retrieval

GMR unifies three retrieval regimes into one formulation and requires a model to return the complete set of relevant temporal moments, or correctly predict an empty set when the queried event does not exist.

  • Supports no / one / multiple moments
  • Combines rejection and localization in one setting
  • Better reflects realistic video retrieval demands

Conventional VMR

  • Assumes exactly one target moment
  • Usually ignores null-set queries
  • Under-explores repeated events

GMR

  • Handles absent, single, and repeated events
  • Measures rejection and localization jointly
  • Expands the scope of temporal grounding research

Three Scenarios

Three Retrieval Scenarios in One Unified Task

01

Null-set Rejection

The query is meaningful but no corresponding event occurs in the video, so the correct output is an empty set.

02

Single-Moment Retrieval

The query corresponds to exactly one temporal segment, aligning with the classic VMR setup.

03

Multi-Moment Retrieval

The same query may occur multiple times, and all relevant moments should be retrieved rather than only the strongest one.

Methods

Methods for Generalized Moment Retrieval

We study GMR from two complementary modeling paradigms: lightweight adaptation of discriminative VMR models and reward design for generative MLLMs.

GMR Adapter

A lightweight plug-and-play module for discriminative VMR backbones, adding an explicit existence-estimation branch for null-set prediction without redesigning the underlying temporal grounding backbone.

  • Compatible with mainstream VMR architectures
  • Explicit null-set awareness
  • Strong end-to-end GMR gains with minimal intrusion

GMR-tailored GRPO Reward

A reward formulation for GRPO-based fine-tuning of generative MLLMs, jointly encouraging temporal localization quality and correct rejection behavior.

  • Targets generalized retrieval objectives
  • Encourages both precision and abstention
  • Extends GMR to generative video-language models
